## How to Use This Toolkit

### 1. Starting a New Project?

1. Copy [BRD Template](templates/brd-template.md)
2. Fill in project-specific details
3. Check [Definition of Ready](checklists/definition-of-ready.md) before dev

### 2. Writing User Stories?

1. Reference [User Stories Library](templates/user-stories-library.md) for examples
2. Use [User Story Card](templates/user-story-card.md) format
3. Validate against [Definition of Ready](checklists/definition-of-ready.md)

### 3. Documenting a Feature?

1. Use [FSD Template](templates/fsd-template.md) for detailed specs
2. Include acceptance criteria
3. Reference [Test Case Template](templates/test-case.md) for QA

### 4. Building AI Features?

1. Review [AI Ethics Checklist](checklists/ai-ethics-checklist.md)
2. Document bias mitigation strategies
3. Plan for transparency and explainability

## Quick Reference Cards

### INVEST Criteria (User Stories)

- **I**ndependent - Can be developed separately
- **N**egotiable - Not a contract, open to discussion
- **V**aluable - Delivers value to user
- **E**stimable - Team can estimate effort
- **S**mall - Fits in one sprint
- **T**estable - Clear pass/fail criteria

### RICE Prioritization

```
RICE Score = (Reach √ó Impact √ó Confidence) / Effort
```

### MoSCoW Prioritization

- **M**ust Have - Critical, non-negotiable
- **S**hould Have - Important, not critical
- **C**ould Have - Nice-to-have
- **W**on't Have - Out of scope (this release)
